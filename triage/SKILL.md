# Triage

Triage issues through a state machine with two category roles and five state roles.

The issue tracker and triage label vocabulary should have been provided to you — run `/setup-matt-pocock-skills` if not.

## Roles

**Category roles** (exactly one per issue):
- `bug` — something broken
- `enhancement` — new capability

**State roles** (exactly one per issue):
- `needs-triage` — not yet evaluated
- `needs-info` — waiting on reporter input
- `ready-for-agent` — autonomous agent can pick this up
- `ready-for-human` — requires human judgment
- `wontfix` — won't be addressed

## Critical rule

**Every comment or issue posted to the issue tracker during triage must start with:**
> *This was generated by AI during triage.*

## Workflow

### 1. Gather context

Read the full issue, all prior triage notes, and relevant codebase areas. Respect prior triage notes — do not re-ask questions already resolved.

### 2. Recommend

Suggest category and state with clear reasoning. Await maintainer direction before applying.

### 3. Reproduce (bugs)

Attempt to verify the reported problem before proceeding to `ready-for-agent`. An unverified bug brief is unreliable.

### 4. Grill if needed

Run a `/grill-with-docs` session to flesh out details when the issue is underspecified.

### 5. Apply outcome

**→ `ready-for-agent`**: Post an agent brief with reproduction steps, relevant code context, and acceptance criteria.

**→ `ready-for-human`**: Explain specifically why human judgment is required (architectural decision, design review, policy call).

**→ `needs-info`**: Post a structured comment capturing:
- What is already established (facts, not questions)
- Specific, actionable questions (not open-ended)

**→ `wontfix`**: Close with explanation. For enhancements, document rationale in out-of-scope knowledge base.

## Notes

- Each issue carries exactly one category role and exactly one state role
- Flag conflicting state roles before acting
- Maintainer directives are authoritative — direct state changes can skip the full grilling process with confirmation
- Issues flow: unlabeled → needs-triage → {needs-info, ready-for-agent, ready-for-human, wontfix}
